Pipeline Flow
End-to-End Architecture — Click Each Stage
01
IMDb Website → Your Laptop
Download IMDb Datasets
title.basics.tsv.gz (800MB) · title.ratings.tsv.gz (25MB)
FREE

Source

IMDb publishes free datasets at datasets.imdbws.com — updated daily. No API key, no signup needed.

Files

title.basics — 12.3M+ titles, all metadata. title.ratings — 1.64M records with averagerating + numvotes.

Format

Tab-separated TSV compressed GZIP. IMDb uses literal \N instead of NULL — requires special handling in pipeline.

"We used IMDb's free public dataset — 12.3M+ records updated daily. The \N null convention required special replacement logic that standard parsers miss."

02
Python boto3 → Amazon S3
upload_to_s3.py — Raw Zone Ingestion
Date-partitioned upload → s3://imdb-pipeline-vikas/raw/imdb/YYYY/MM/DD/
FREE

Script

upload_to_s3.py uses boto3.upload_file() with progress logging. Reads from local data/ folder.

S3 Path

s3://imdb-pipeline-vikas/raw/imdb/2026/03/02/ — date partition isolates each batch for reprocessing any historical date.

Cost

S3 free tier: 5GB storage. Two files = ~825MB = $0.02/month. PUT requests = fractions of a cent.

"Raw files land in date-partitioned S3 — we can always reprocess any day by pointing the pipeline at a specific partition without touching other data."

03
Amazon EMR 7.2.0 · m5.xlarge × 2
imdb_spark_etl.py — PySpark Distributed ETL
Read → Clean \N → Filter movies → Join → DQ → Write Parquet
~$0.10

Step 1 — Read

Spark reads TSV.GZ directly from S3 into distributed DataFrames across cluster nodes. No local files needed.

Step 2 — Clean

Replace \N with null using Spark when(col == "\\N", None). Filter titletype == "movie" only.

Step 3 — Join

Inner join basics + ratings on tconst. Only movies with both metadata AND ratings kept — 738K → 339K.

Step 4 — DQ

4 rules as boolean mask: rating 1-10, votes > 0, title not null, tconst not null. Split into passed/failed DataFrames.

Step 5 — Write

Passed → Snappy Parquet to processed/movies/. Failed → JSON to quarantine/movies/.

Cluster Config

EMR 7.2.0, Spark + Hadoop. m5.xlarge × 2 On-Demand nodes. Terminated immediately after job = $0.10 total.

"PySpark on EMR distributes the 12.3M record job across 2 nodes — each executor processes a partition independently. When data grows to 100GB+, we scale nodes without changing code."

04
Amazon S3 — Processed Zone
Snappy Parquet Output
part-00000.snappy.parquet → columnar, 10x cheaper for Athena
~$0.02/mo

Why Parquet

Columnar storage — Athena reads only requested columns. 10x less data scanned = 10x cheaper than CSV.

Snappy compression

~60% smaller than uncompressed. Fast decompression. Industry standard for Spark/Athena analytical workloads.

3-Zone Architecture

raw/ landing zone · processed/ serving zone · quarantine/ investigation zone. Clear data lineage at each stage.

"Writing Parquet instead of CSV reduced Athena query costs by 10x — Athena charges per TB scanned and columnar format only reads relevant columns."

05
AWS Glue
Glue Crawler → Data Catalog
Auto-detects Parquet schema → updates Hive Metastore
~$0.01

Glue Crawler

Scans s3://imdb-pipeline-vikas/processed/movies/ after each ETL run — auto-detects all column names, types, and partitions.

Glue Data Catalog

Acts as central Hive Metastore — both EMR Spark and Athena share schema definitions. Single source of truth.

Auto-triggered

PySpark job calls glue.start_crawler() via boto3 after writing Parquet — zero manual intervention needed.

"Glue Data Catalog is the Hive Metastore of our data lake — schema defined once, shared across all query engines. Adding Redshift Spectrum later requires zero schema migration."
↓
06
Amazon Athena
Serverless SQL Analytics
External table on S3 Parquet → top movies, genre trends, decade analysis
~$0.001

External Table

Athena doesn't store data — reads directly from S3 Parquet via Glue Catalog schema. Zero ETL into separate database.

Query Cost

$5 per TB scanned. IMDb Parquet = ~100MB. Each query = ~$0.001. Running 100 queries/month = $0.10 total.

Sample Queries

Top 10 movies by rating, movies per decade, genre popularity, rating distribution — all run in 2-3 seconds.

"Athena external tables give us serverless SQL on 339K movie records with zero infrastructure — queries run in 2-3 seconds at $0.001 each."

Grafana + Docker (localhost:3000)
Analytics Dashboard
4 panels connected to Athena → Top movies · Genres · Decades · Ratings
FREE

Setup

Grafana runs via Docker on localhost:3000. Amazon Athena plugin connects to AWS with access keys. Queries run directly against S3 Parquet.

4 Panels

Top 10 highest rated movies (bar). Movies per decade (bar). Genre distribution (pie). Rating distribution (bar).

Why Grafana

Free forever, open source, more chart types than QuickSight. Dashboard JSON stored in GitHub — fully portable.

"Grafana on Docker connects directly to Athena — no data movement, no extra storage. Dashboard JSON lives in our GitHub repo for reproducibility."
↓
08
EventBridge + Lambda + SNS
Automated Daily Scheduling
Cron: 0 1 * * ? * → Lambda → Glue Crawler → SNS email
FREE

EventBridge

Cron rule 0 1 * * ? * fires daily at 1 AM UTC — completely serverless, no EC2 needed. Free tier: 14M events/month.

Lambda

Python 3.12 function triggers Glue Crawler via boto3, sends SNS notification with run status and metrics.

SNS Alerts

3 alert types: ✅ SUCCESS with DQ metrics, ❌ FAILURE with stack trace, ⚠️ WARNING if DQ rate drops below 95%.

"EventBridge + Lambda gives us zero-infrastructure scheduling — no Airflow server, no EC2, no maintenance. The entire orchestration layer costs $0/month."
